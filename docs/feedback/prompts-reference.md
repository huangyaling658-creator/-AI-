# 反馈分析 Prompt 参考文档

> 本文档记录反馈分析模块使用的所有 Prompt，便于后续调优和复用。
> 源码位置：`src/lib/feedback/prompts.ts`

---

## 1. 维度推荐 Prompt（buildDimensionPrompt）

**用途：** 上传数据后，AI 根据样本推荐 3-6 个分析维度

**输入：** 前 10 条反馈样本

**Prompt 全文：**

```
你是一位资深用户研究专家。以下是用户上传的反馈数据样本（共{N}条）。

请分析这些数据的特征，推荐 3-6 个最有价值的分析维度。

对每个维度说明：
1. name: 维度名称（简短，如"情感倾向"、"问题分类"）
2. description: 一句话说明分析这个维度的价值
3. supported: true/false，当前数据是否支持此维度
4. reason: 为什么支持或不支持

常见维度供参考（不限于此，根据数据特征灵活推荐）：
- 情感倾向：正面/负面/中立分布
- 问题分类：功能缺陷、体验问题、性能问题、需求建议等
- 优先级评估：按频次和情感强度综合评分
- 功能模块归属：反馈涉及产品的哪个模块/功能
- 用户画像标签：如果数据含用户信息
- 时间趋势：如果数据含时间信息

严格要求：
1. 只返回纯 JSON 数组，禁止用 markdown 代码块（```）包裹
2. 禁止在 JSON 前后添加任何解释文字
3. 字符串值中不要包含换行符，用空格替代
4. 示例格式：[{"name":"情感倾向","description":"分析正面负面中立分布","supported":true,"reason":"数据包含明确态度表达"}]

## 反馈样本数据
1. {反馈1}
2. {反馈2}
...
```

**期望输出格式：**
```json
[
  {"name": "情感倾向", "description": "分析正面负面中立分布", "supported": true, "reason": "数据包含明确态度表达"},
  {"name": "问题分类", "description": "按功能缺陷/体验/性能/建议分类", "supported": true, "reason": "反馈涉及多种问题类型"}
]
```

## 2. 深度分析 Prompt（buildAnalysisPrompt）

**用途：** 对全量反馈数据进行分类、情感分析、关键词提取等

**输入：** 全部反馈文本 + 用户选择的分析维度

**Prompt 全文：**

```
你是一位资深用户研究专家，请对以下用户反馈数据进行深度分析。

## 数据清洗规则（分析前先执行）
1. 去除完全重复的反馈
2. 过滤无意义内容（纯表情、单字回复如"好""嗯""OK"、无关广告等）
3. 合并表述不同但含义相同的反馈（保留一条，计入频次）

## 分析维度
1. {维度1}
2. {维度2}
...

## 输出要求（严格遵守）
1. 只返回纯 JSON 对象，禁止用 markdown 代码块包裹
2. 禁止在 JSON 前后添加任何解释文字
3. 所有字符串值中禁止包含换行符，用空格替代
4. 所有字符串值中的双引号用中文引号""替代
5. topIssues 最多 10 条，keywords 最多 20 个，均按 count 降序
6. feedbacks 数组中每条的 content 字段最多保留前 100 个字符
7. percent 保留 1 位小数，分类名称使用中文

JSON 结构：
{"summary":{"total":数字,"valid":数字,"deduplicated":数字},"categories":[{"name":"分类名","count":数字,"percent":数字,"feedbackIds":[0,3,7]}],"sentiments":{"positive":{"count":数字,"percent":数字},"negative":{"count":数字,"percent":数字},"neutral":{"count":数字,"percent":数字}},"topIssues":[{"issue":"问题描述","count":数字,"sentiment":"negative","keywords":["词1","词2"]}],"keywords":[{"word":"关键词","count":数字}],"feedbacks":[{"id":0,"content":"原文前100字","category":"分类","sentiment":"negative","sentimentScore":-0.8,"keywords":["词"]}]}

## 反馈数据（共{N}条）
[0] {反馈内容1}
[1] {反馈内容2}
...
```

**期望输出字段说明：**

| 字段 | 说明 |
|------|------|
| summary.total | 原始总条数 |
| summary.valid | 清洗后有效条数 |
| summary.deduplicated | 去重后条数 |
| categories[] | 分类统计，含名称、数量、百分比、原始ID列表 |
| sentiments | 情感分布（正面/负面/中立） |
| topIssues[] | 高频问题 Top10，含关键词 |
| keywords[] | 高频关键词 Top20 |
| feedbacks[] | 每条反馈的标注结果（分类+情感+关键词） |

## 3. 多文件字段映射 Prompt（buildMergeFieldPrompt）

**用途：** 用户上传多份文件时，识别每份文件中哪一列是反馈内容

**Prompt 全文：**

```
你是数据分析专家。以下是用户上传的多份反馈数据文件，每份文件有不同的列名。

请识别每份文件中哪一列最可能是"用户反馈内容"（如评论、意见、建议等）。

严格以 JSON 数组格式返回，不要包含其他文字：
[{"fileName":"文件名","feedbackColumn":"列名"}]

## 文件信息
### {文件名}
列名: col1, col2, col3
示例数据:
row1_col1 | row1_col2 | row1_col3
row2_col1 | row2_col2 | row2_col3
```

## 4. 图片 OCR Prompt（buildImageOCRPrompt）

**用途：** 从上传的图片中识别并提取反馈文本

**Prompt 全文：**

```
请识别这张图片中的所有文字内容。如果图片包含用户反馈、评论或意见类的内容，请逐条提取。

以 JSON 数组格式返回每条反馈文本：
["反馈1", "反馈2", ...]

如果图片中没有可识别的反馈内容，返回空数组 []
```

## 分析方法论

采用 **语义聚类 + 情感分析 + 频次统计** 三合一方案：

1. **语义聚类** — 模型理解语义后按相似性归组，发现隐性主题
2. **情感分析** — 每条反馈打情感分（-1 到 +1），量化用户态度
3. **频次统计** — 相似问题出现次数统计，找高频痛点
4. **数据清洗** — 去重、过滤无意义内容、合并同义表达
5. **摘要提炼** — 每个类别的 topIssues 生成一句话摘要

## 性能参数

| 参数 | 维度推荐 | 深度分析 |
|------|----------|----------|
| 样本量 | 前 10 条 | 全量（>300条分批） |
| maxTokens | 1024 | 8192 |
| temperature | 0.3 | 0.2 |
| 超时 | 30s | 120s |
| 分批大小 | - | 300 条/批 |
